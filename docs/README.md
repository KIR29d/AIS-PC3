## Содержание документов

### 1. Техническое задание (ТЗ)
**Файл:** `specifications/ТЗ_ГОСТ_34.602-2020.md`

Документ разработан в соответствии с ГОСТ 34.602-2020 и содержит:
- Общие сведения о системе
- Назначение и цели создания
- Характеристику объекта автоматизации
- Требования к системе
- Состав и содержание работ по созданию
- Порядок контроля и приемки
- Требования к документированию

### 2. Диаграммы

#### ER-диаграмма
**Файл:** `diagrams/er_diagram.puml`

Отображает структуру базы данных:
- 12 таблиц со всеми полями и типами данных
- Первичные и внешние ключи
- CHECK ограничения
- Отношения между таблицами

#### Диаграмма вариантов использования
**Файл:** `diagrams/use_case_diagram.puml`

Показывает взаимодействие акторов с системой:
- Акторы: Администратор, Менеджер, Сборщик, Клиент, Веб-пользователь
- 28 вариантов использования
- Отношения include/extend

#### Диаграмма классов
**Файл:** `diagrams/class_diagram.puml`

Описывает объектную модель системы:
- Классы сущностей (Entity)
- Классы сервисов (Service)
- Интерфейсы
- Отношения между классами

#### Диаграмма компонентов
**Файл:** `diagrams/component_diagram.puml`

Архитектура системы:
- Desktop приложение (WPF)
- Business Logic Layer
- Data Access Layer
- Database (MS SQL Server)
- Web приложение (перспектива)

#### Диаграмма коммуникации
**Файл:** `diagrams/communication_diagram.puml`

Взаимодействие объектов при создании заказа:
- Нумерация сообщений
- Объекты: формы, контроллеры, репозитории, сущности

#### Диаграммы последовательности

**Создание заказа** (`diagrams/sequence_order_creation.puml`):
- Поиск/регистрация клиента
- Оформление заказа
- Добавление компонентов
- Создание сборочного задания
- Назначение исполнителя

**Процесс сборки** (`diagrams/sequence_assembly_process.puml`):
- Начало работы со сборочным заданием
- Сканирование компонентов
- Резервирование на складе
- Завершение сборки
- Формирование отчета

**Аутентификация** (`diagrams/sequence_authentication.puml`):
- Вход в систему
- Проверка учетных данных
- Инициализация интерфейса по роли
- Контроль доступа
- Выход из системы

## Как использовать

### Просмотр диаграмм PlantUML

1. **Онлайн:**
   - Откройте https://www.plantuml.com/plantuml/
   - Скопируйте содержимое .puml файла
   - Вставьте и получите визуализацию

2. **VS Code:**
   - Установите расширение "PlantUML"
   - Откройте .puml файл
   - Нажмите Alt+D для предпросмотра

3. **Локально:**
   ```bash
   java -jar plantuml.jar diagram.puml
   ```

### Развертывание базы данных

1. Откройте SQL Server Management Studio или Azure Data Studio
2. Подключитесь к экземпляру MS SQL Server
3. Откройте файл `database/create_database.sql`
4. Выполните скрипт (F5)

### Редактирование Mind Map

Mind Map представлен в формате Markdown с иерархической структурой.
Для визуального представления можно использовать:
- Markmap (https://markmap.js.org/)
- XMind (импорт Markdown)
- MindMeister

## Роли пользователей

| Роль | Описание | Права доступа |
|------|----------|---------------|
| Администратор | Управление системой | Полный доступ |
| Менеджер | Работа с клиентами и заказами | Заказ, клиенты, сборка |
| Сборщик | Сборка ПК | Сборочные задания, склад |
| Клиент | Заказчик услуг | Личный кабинет, заказы |
