# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Автоматизированная информационная система для магазина кастомных ПК

### 1. ОБЩИЕ СВЕДЕНИЯ

#### 1.1 Наименование системы
Автоматизированная информационная система «CustomPC Manager» (далее – АИС).

#### 1.2 Область применения
АИС предназначена для автоматизации бизнес-процессов магазина кастомных персональных компьютеров, включая управление заказами, сборкой ПК, учетом компонентов на складе, взаимодействием с клиентами и сотрудниками.

### 2. НАЗНАЧЕНИЕ И ЦЕЛИ СОЗДАНИЯ СИСТЕМЫ

#### 2.1 Назначение системы
АИС предназначена для:
- Управления базой клиентов (физические и юридические лица)
- Оформления и отслеживания заказов на сборку ПК
- Управления процессом сборки компьютеров
- Учета компонентов на складах
- Контроля работы сотрудников и отделов
- Ведения аудита изменений в системе
- Предоставления веб-интерфейса для покупателей (в перспективе)

#### 2.2 Цели создания системы
- Повышение эффективности обработки заказов
- Снижение количества ошибок при сборке ПК
- Оптимизация складского учета
- Улучшение качества обслуживания клиентов
- Обеспечение прозрачности бизнес-процессов

---

### 3. ХАРАКТЕРИСТИКА ОБЪЕКТА АВТОМАТИЗАЦИИ

#### 3.1 Краткие сведения об объекте
Объектом автоматизации является магазин кастомных персональных компьютеров, осуществляющий:
- Прием заказов от клиентов на сборку ПК по индивидуальным конфигурациям
- Закупку и хранение комплектующих
- Сборку и тестирование компьютеров
- Отгрузку готовых изделий клиентам

#### 3.2 Условия функционирования
- Режим работы: круглосуточный (для серверной части), 9:00–20:00 (для рабочих мест сотрудников)
- Количество одновременных пользователей: до 50 человек
- Требования к доступности: 99% в рабочее время

#### 3.3 Существующие бизнес-процессы
- Оформление заказов через менеджеров
- Ручное ведение учета компонентов
- Бумажное документооборот по сборочным заданиям

### 4. ТРЕБОВАНИЯ К СИСТЕМЕ

#### 4.1 Требования к структуре и функционированию

##### 4.1.1 Подсистемы АИС
1. **Подсистема управления пользователями**
   - Регистрация и аутентификация пользователей
   - Управление ролями и правами доступа
   - Ведение журнала входов в систему

2. **Подсистема управления клиентами**
   - Регистрация клиентов (физических и юридических лиц)
   - Хранение контактной информации
   - История заказов клиента

3. **Подсистема управления сотрудниками**
   - Учет сотрудников и их должностей
   - Привязка к отделам
   - Назначение сборочных заданий

4. **Подсистема управления заказами**
   - Создание и редактирование заказов
   - Отслеживание статуса заказа
   - Расчет общей суммы заказа

5. **Подсистема управления сборкой**
   - Создание сборочных заданий
   - Назначение исполнителей
   - Контроль этапов сборки
   - Фиксация используемых компонентов

6. **Подсистема складского учета**
   - Учет компонентов по складам
   - Контроль остатков
   - Обновление информации о наличии

7. **Подсистема аудита**
   - Логирование всех изменений в системе
   - Хранение истории операций
   - Возможность формирования отчетов

8. **Веб-подсистема для покупателей** (перспектива)
   - Просмотр статусов заказов
   - Конфигуратор ПК
   - Личный кабинет клиента

##### 4.1.2 Функциональные требования
- Система должна обеспечивать разграничение прав доступа на основе ролей
- Все критические операции должны протоколироваться в таблице аудита
- Статусы заказов и сборок должны изменяться согласно утвержденному workflow
- Система должна поддерживать работу с несколькими складами
- Должна быть обеспечена целостность данных через внешние ключи и ограничения

#### 4.2 Требования к численности и квалификации персонала
- Администратор системы: 1–2 человека с навыками администрирования СУБД
- Пользователи (менеджеры, сборщики): без специальных требований к квалификации
- Техническая поддержка: 1–2 человека со знанием C# и MS SQL Server

#### 4.3 Требования к надежности
- Резервное копирование базы данных: ежедневно
- Время восстановления после сбоя: не более 4 часов
- Защита от несанкционированного доступа: аутентификация, авторизация, аудит
